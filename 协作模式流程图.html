四角色协作分工

👩‍⚕️ 社工（主力）

15人，每人负责若干老人，7×24响应

  • 接收全部等级告警
  • 确认处理 / 标记安全
  • 转派物业上门
  • 标记误报 + 填原因
  • 管理老人档案

👴 楼长

每栋楼1位，年龄偏大，适老化界面

  • 只看本楼栋告警
  • 看不到敏感信息（脱敏）
  • 辅助确认老人状态
  • 大字模式 + 语音播报

🏢 物业

物业值班人员，负责紧急上门

  • 仅看 P0 紧急告警
  • 接受社工转派
  • 到场后回填结果
  • 无档案写入权限

📋 主任

社区主任，全局监督

  • 全部告警只读
  • 升级通知最终接收人
  • 查看统计看板
  • 审核误报数据

P0 紧急告警升级机制

0 ~ 5分钟
责任社工

5分钟未响应

@手机号 + 语音双呼
10分钟未响应

全群通知 + 主任电话

告警状态流转

待处理 → 处理中 → 已关闭
处理中 → 误报关闭
处理中 → 已派单 → 已关闭

一次完整的 P0 告警响应（示例）

02:15 — 传感器触发
张大爷卧室床垫传感器检测到离床 > 30分钟，同时卫生间PIR有人体存在
02:15 — 规则引擎判定 R-MIX-01 → P0
夜间 + 离床超时 + 厕所有人 = 疑似跌倒，生成 P0 紧急告警
02:15 — 钉钉推送
互动卡片推送至工作群，同时语音双呼责任社工李姐
02:17 — 社工确认
李姐点击卡片「确认处理」，告警进入处理中状态
02:18 — 转派物业
李姐判断需要上门，点击「转派」，物业值班收到通知
02:25 — 物业到场
物业值班到场确认张大爷安全（起夜后在卫生间休息），回填「到场确认安全」
02:25 — 告警关闭
全程 10 分钟内响应完毕，记录归档
